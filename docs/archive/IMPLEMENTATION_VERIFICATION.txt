STREAK CALCULATION
- Examines reading-dates object
- Counts consecutive days from today backwards
- Handles date changes correctly (UTC consideration)
- Returns total consecutive days
- Resets on any break in daily reading

AUTOMATIC TRACKING
- trackReadingSession() called when guide marked complete
- Logs date in reading-dates object
- Checks for daily and streak achievements
- Unlocks achievements automatically
- Shows toast notification on unlock

CATEGORY TRACKING
- Reads data-tags attribute from guide cards
- Tracks which categories have been read
- Counts unique categories for Explorer achievement
- Categorizes guides for category-master achievements

================================================================================
STORAGE STRUCTURE
================================================================================

localStorage Keys Used:
1. "achievements" 
   - Format: { "achievement-id": { unlockedAt: timestamp }, ... }
   - Contains unlock time for each achievement

2. "achievement-reading-dates"
   - Format: { "Day Month DD YYYY": [guideId1, guideId2, ...], ... }
   - Records which guides read on each date

3. "current-reading-streak"
   - Format: integer (number of days)
   - Cached for fast access

4. "compendium-progress"
   - Format: { guideId: { completed: true, notes: "..." }, ... }
   - Existing progress tracking (not new)
